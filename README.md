# LLM FHIR Query Evaluation

This repository contains a collection of prompts and test data designed to evaluate the performance of Large Language Models (LLMs) in generating proper FHIR (Fast Healthcare Interoperability Resources) queries.

## Purpose

The goal of this project is to systematically assess how well different LLMs can:
- Understand healthcare data requirements expressed in natural language
- Generate syntactically correct FHIR queries
- Produce semantically accurate queries that retrieve the intended healthcare data
- Handle complex clinical scenarios and edge cases

## Contents

- **Prompts**: A curated set of natural language requests for healthcare data queries
- **Test Data**: Sample FHIR resources and expected query results for validation
- **Evaluation Metrics**: Criteria for assessing LLM-generated FHIR queries

## About FHIR

FHIR is a standard for exchanging healthcare information electronically. It defines a set of resources (like Patient, Observation, Medication) and APIs for querying and manipulating healthcare data.

## Usage

This repository serves as a benchmark for testing and comparing LLM capabilities in the healthcare data querying domain, particularly for applications requiring automated FHIR query generation from natural language inputs.
